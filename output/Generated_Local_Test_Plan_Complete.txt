### Test 1
Coverage for item: [SW-207464]:CLI-request | request ethernet-oam cfm on-demand stop

1. **Test Name:** PM Initiator | Two-way Delay Measurement with MEP ID  
2. **Test Description:** Validate proactive initiator two-way delay measurement (ETH-DM) between DNOS devices using MEP ID targeting and verify delay and jitter statistics.  
3. **Test Steps:**  
   1. On DUT, configure CFM MEP under sub-interface ge100-0/0/0.103:  
      ```
      interface ge100-0/0/0.103
       encapsulation dot1q 103
       ethernet cfm mep domain MD1 maintenance-association MA1 mep-id 1
      ```  
   2. On DN01, configure CFM MEP under sub-interface ge100-0/0/0.203:  
      ```
      interface ge100-0/0/0.203
       encapsulation dot1q 203
       ethernet cfm mep domain MD1 maintenance-association MA1 mep-id 2
      ```  
   3. On DUT, create and enable a two-way delay measurement session named dm1:  
      ```
      services performance-monitoring cfm two-way-delay-measurement dm1
       source maintenance-domain MD1 maintenance-association MA1 mep-id 1
       target mep-id 2 pcp 3 admin-state enabled
      ```  
   4. Verify the session configuration on DUT:  
      ```
      show services performance-monitoring cfm two-way-delay-measurement dm1
      ```  
   5. Verify measurement statistics after at least three probes:  
      ```
      show services performance-monitoring cfm two-way-delay-measurement dm1 statistics
      ```  
   6. Disable and remove the session on DUT:  
      ```
      services performance-monitoring cfm two-way-delay-measurement dm1 admin-state disabled
      no services performance-monitoring cfm two-way-delay-measurement dm1
      ```  
4. **Pass Criteria:**  
   1. `show ethernet cfm mep domain MD1 maintenance-association MA1` on DUT under ge100-0/0/0.103 shows MEP-ID 1.  
   2. `show ethernet cfm mep domain MD1 maintenance-association MA1` on DN01 under ge100-0/0/0.203 shows MEP-ID 2.  
   3. DUT accepts the two-way-delay-measurement dm1 configuration and admin-state is enabled.  
   4. `show services performance-monitoring cfm two-way-delay-measurement dm1` lists session dm1 with correct source MD1/MA1 MEP-ID 1, target MEP-ID 2, PCP 3 and admin-state enabled.  
   5. Statistics output shows DMM PDUs transmitted ≥3, Valid DMR PDUs received ≥1, and round-trip-delay and jitter values present.  
   6. Session dm1 admin-state shows disabled and `show services performance-monitoring cfm two-way-delay-measurement` no longer lists dm1.  
5. **Variants:**  
   - Use `target mac-address <MAC>` instead of `target mep-id`.  
   - Perform one-way delay measurement (`one-way` option) instead of two-way.  
   - Change PCP value to other 802.1p priorities (0–7).  
   - Run the same test on a double-tagged VLAN sub-interface (e.g., `.100.103`).

### Test 5
Coverage for item: [SW-206837]:CLI-show | show services performance-monitoring connectivity-fault-management sessions

1. **Test Name:** PM Initiator | Two-Way Delay Measurement Unicast  
2. **Test Description:** Verify that DUT can initiate a Y.1731 two-way delay measurement (ETH-DM) session using a profile, target a remote MEP by MAC and by MEP-ID, collect statistics, and properly disable the session.  
3. **Test Steps:**  
   1. On DUT, configure a two-way delay measurement profile PROFILE_DM:  
      ```
      config
        services performance-monitoring profiles cfm two-way-delay-measurement PROFILE_DM
          inform-test-results enabled
          test-duration probe-count 5 probe-interval 1 repeat-interval 5
          thresholds delay-rtt-max 5000 jitter-rtt-max 2000 success-rate 90
      commit
      ```  
   2. On DN01, configure a remote CFM MEP (MD1/MA1, mep-id 2) on the link to DUT (interface ge100-0/0/0.203):  
      ```
      config
        services cfm maintenance-domain MD1
          maintenance-association MA1
            mep 2 interface ge100-0/0/0.203
      commit
      ```  
   3. On DUT, create and enable session SESSION_MAC on the DUT→DN01 link (interface ge100-0/0/0.204) targeting remote MAC 00:11:22:33:44:55 with PCP=3:  
      ```
      config
        services performance-monitoring cfm two-way-delay-measurement SESSION_MAC
          interface ge100-0/0/0.204
          admin-state enabled
          profile PROFILE_DM
          source maintenance-domain MD1 maintenance-association MA1 mep-id 1
          target mac-address 00:11:22:33:44:55 pcp 3
      commit
      ```  
   4. On DUT, show active DM sessions and verify SESSION_MAC is “up”:  
      ```
      show services performance-monitoring cfm two-way-delay-measurement sessions
      ```  
   5. On DUT, show statistics for SESSION_MAC and verify 5 probes sent, ≥1 reply received:  
      ```
      show services performance-monitoring cfm two-way-delay-measurement statistics SESSION_MAC
      ```  
   6. On DUT, disable and remove SESSION_MAC, then verify it no longer appears:  
      ```
      config
        services performance-monitoring cfm two-way-delay-measurement SESSION_MAC admin-state disabled
      commit
      show services performance-monitoring cfm two-way-delay-measurement sessions
      ```  
   7. On DUT, create and enable session SESSION_ID on the same interface targeting mep-id 2 with PCP=4:  
      ```
      config
        services performance-monitoring cfm two-way-delay-measurement SESSION_ID
          interface ge100-0/0/0.204
          admin-state enabled
          profile PROFILE_DM
          source maintenance-domain MD1 maintenance-association MA1 mep-id 1
          target mep-id 2 pcp 4
      commit
      ```  
   8. On DUT, show statistics for SESSION_ID and verify 5 probes sent, ≥1 reply received:  
      ```
      show services performance-monitoring cfm two-way-delay-measurement statistics SESSION_ID
      ```

   1. PROFILE_DM appears under `show services performance-monitoring profiles cfm two-way-delay-measurement PROFILE_DM`.
